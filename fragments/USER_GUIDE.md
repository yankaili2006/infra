# Fragments 会话历史功能使用指南

## 🚀 快速开始

### 访问应用
```
http://localhost:3000
或
http://100.64.0.23:3000
```

---

## 📖 主要功能

### 1. 打开会话历史侧边栏

点击导航栏右上角的 **History** 图标（📜）

### 2. 创建新会话

- 点击侧边栏顶部的"新建对话"按钮
- 或直接开始输入消息（自动创建会话）

### 3. 查看历史会话

- 侧边栏显示所有保存的会话
- 每个会话显示：标题、预览、时间、消息数量

### 4. 切换会话

- 点击任意历史会话即可加载
- 当前会话会高亮显示

### 5. 管理会话

**重命名**：
- 鼠标悬停在会话上
- 点击编辑图标（✏️）
- 输入新标题并确认

**删除**：
- 鼠标悬停在会话上
- 点击删除图标（🗑️）
- 确认删除操作

---

## 💡 使用技巧

### 自动保存
- 所有消息自动保存到当前会话
- 无需手动保存

### 会话标题
- 首次发送消息时自动生成标题
- 可随时重命名

### 会话排序
- 按最后更新时间排序
- 最近使用的会话在最上方

---

## 🎯 常见场景

### 场景 1: 开始新项目
1. 点击"新建对话"
2. 输入项目相关问题
3. 会话自动保存

### 场景 2: 继续之前的工作
1. 打开会话历史
2. 找到相关会话
3. 点击加载并继续

### 场景 3: 整理会话
1. 重命名会话为有意义的标题
2. 删除不需要的会话
3. 保持会话列表整洁

## ⚙️ 技术说明

### 数据存储
- 会话保存在 `.sessions/` 目录
- 每个会话一个 JSON 文件
- 支持备份和迁移

### 会话格式
```json
{
  "id": "session_xxx",
  "title": "会话标题",
  "messages": [...],
  "createdAt": "2026-02-04T...",
  "updatedAt": "2026-02-04T..."
}
```

## 🔧 故障排除

### 问题：会话列表为空
- 检查是否已发送消息
- 刷新页面重新加载

### 问题：无法加载会话
- 检查 `.sessions/` 目录权限
- 查看浏览器控制台错误

### 问题：会话未保存
- 确认已发送消息
- 检查 API 是否正常工作
